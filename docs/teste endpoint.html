Como testar endpoints protegidos no Swagger

Pré-requisitos

  • API rodando com um perfil que tenha RoleSystem configurado (ex: LOCAL_LDT_ACP ou LOCAL_LDT_PRD)
  • Credenciais LDAP válidas (usuário de rede Scania)

Passo 1 — Gerar a senha em Base64

O endpoint POST /api/User/login exige que a senha seja enviada em Base64.

Opção A — PowerShell

[Convert]::ToBase64String([Text.Encoding]::UTF8.GetBytes("sua.senha"))

Opção B — Terminal (Linux/macOS)

echo -n "sua.senha" | base64

Opção C — C# (snippet rápido)

var senha = "sua.senha";
var base64 = Convert.ToBase64String(System.Text.Encoding.UTF8.GetBytes(senha));
Console.WriteLine(base64);

Opção D — Site online

Acesse https://www.base64encode.org, cole sua senha e clique em Encode.

⚠️ Importante: Apenas a senha precisa estar em Base64. O UserName é enviado em texto simples.

Passo 2 — Fazer login e obter o token JWT

  1. Acesse o Swagger em http://localhost:8080 (ou https://localhost:5001)
  2. Localize o endpoint POST /api/User/login
  3. Clique em Try it out
  4. Preencha o body:
{
  "userName": "seu.usuario",
  "password": "c3VhLnNlbmhh"
}
  1. Clique em Execute
  2. Na resposta, copie o valor do campo token:
{
  "content": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  }
}

⚠️ O token expira em 1 hora. Após esse tempo, repita este passo.

Passo 3 — Autorizar o Swagger com o token

  1. Clique no botão 🔒 Authorize no topo da página do Swagger
  2. No campo Value, cole somente o token (sem o prefixo Bearer):
eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
  1. Clique em Authorize e depois em Close

O cadeado 🔒 ficará fechado, indicando que todas as requisições passarão a incluir o header:

Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

Passo 4 — Testar o endpoint GetEwoTaskRowByFilter

  1. Localize GET /api/Parameters/GetEwoTaskRow
  2. Clique em Try it out
  3. Preencha ao menos um dos parâmetros:
Parâmetro Descrição Exemplo
popid Número do POPID do veículo 773561
productIndividual Código do produto individual 11162715
  1. Clique em Execute

Respostas esperadas

HTTP Situação
200 OK Registros encontrados com sucesso
400 Bad Request Nenhum parâmetro informado
401 Unauthorized Token ausente, expirado ou role inválida
404 Not Found Nenhum registro para os filtros informados
429 Too Many Requests Limite de 2 requisições por minuto excedido
